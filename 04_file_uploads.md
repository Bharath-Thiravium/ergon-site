# User Manual: File Uploads

## 4.1 Overview

The File Uploads module provides secure storage and management of documents within the Ergon Site system. Users can upload, organize, and share files related to projects, expenses, and personal documents.

## 4.2 Supported File Types

- Documents: PDF, DOC, DOCX, XLS, XLSX
- Images: JPG, PNG, GIF
- Archives: ZIP (up to 50MB)
- Other: TXT, CSV

## 4.3 Uploading Files

**Procedure:**
1. Navigate to **Files > Upload** in the main menu.
2. Click **Choose Files** or drag and drop files into the upload area.
3. Select the **Category/Folder** (e.g., Personal, Project Documents).
4. Add optional **Description** and **Tags**.
5. Click **Upload**.

**Expected Outcome:**
- Files are stored securely in the cloud.
- Progress bar shows upload status.
- Large files may take time; avoid closing the browser during upload.

## 4.4 File Organization

- **Folders:** Create personal or shared folders.
- **Tags:** Add keywords for easy searching.
- **Categories:** Pre-defined folders for different purposes (HR Documents, Project Files, etc.).

## 4.5 Accessing Uploaded Files

1. Go to **Files > My Files**.
2. Browse folders or use the search bar.
3. Click on a file to view, download, or share.

## 4.6 Sharing Files

1. Select a file from your list.
2. Click **Share**.
3. Enter recipient email addresses.
4. Set permissions: View or Download.
5. Add an optional message.
6. Click **Send**.

## 4.7 File Versioning

- Upload new versions of existing files.
- System maintains version history.
- Access previous versions from file details.

## 4.8 Storage Limits

- Individual files: Up to 50MB
- Total storage: Based on your organization's plan
- Automatic cleanup of temporary files

## 4.9 Security and Privacy

- All files are encrypted at rest and in transit.
- Access controls based on user roles.
- Audit logs track file access and modifications.

## 4.10 Bulk Uploads

For multiple files:
1. Select multiple files during upload.
2. System processes them as a batch.
3. Monitor progress in the uploads queue.

## 4.11 Troubleshooting

- **Upload Fails:** Check file size and type; ensure stable internet.
- **File Not Found:** Verify permissions or contact admin.
- **Corrupt Files:** Re-upload or contact support.

## 4.12 Collaboration Features

- **Comments:** Add notes to files for team discussion.
- **Annotations:** Highlight and comment on PDFs.
- **Real-time Editing:** For supported formats like Google Docs integration.

## 4.13 File Permissions

- **Public:** Accessible to all users.
- **Private:** Only uploader can access.
- **Shared:** Specific users or groups.

## 4.14 Integration with Other Modules

- Attach files to tasks, expenses, or profiles.
- Sync with cloud storage (Google Drive, OneDrive).

## 4.15 File Analytics

View download counts, access logs, and popular files.

## 4.16 Backup and Recovery

Files are backed up daily. Request recovery from admin if needed.

## 4.17 Mobile Access

Upload and access files via mobile app.

## 4.18 API Access

Developers can use APIs for automated file uploads.

## 4.19 Best Practices

- Use descriptive names.
- Organize in folders.
- Regularly review and delete unnecessary files.
- Use tags for better searchability.
